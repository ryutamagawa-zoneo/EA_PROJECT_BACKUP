② 開発運用規約（出力・実装ルール）

（旧：開発運用規約 

開発運用規約

 の統合対応版）

目的

出力形式・手順・統合順を固定し事故を防ぐ。

1. フルコード定義

・1ファイル完結
・省略なし
・コンパイル可能
・修正後全文

満たせない場合は出力禁止。

2. GOフロー

出力可能時のみ
「出力可能です。GOの合図で出力します。」

GO前にコード出力禁止。
GO後は説明なしでフルコード出力。

（エラー修正時はGO不要）

3. 出力形式

コード本文はチャットに貼らない。
必ず .txt で出力。

4. レイヤー識別（統合追加）

フルコード冒頭に明記：

CODE NAME: XXXXX
LAYER: BASE / HL / CL / INT

5. 統合順序（固定）

BASE確定

HL確定

INT接続

動作確認

Compliance接続

順序変更禁止。

6. 統合時の変更区分

ADD(HL)
MODIFY(BASE)
ADD(INT)

DELETE(BASE) 原則禁止。

7. データ不足時

不足があれば進行禁止。
「◯◯のデータを下さい」と明示要求。

（ビルドエラー差分テンプレ・LOG責務等は原規約をそのまま適用）
