2. 純粋憲章（Charter）
EA開発プロジェクト 基本憲章（不変）
0. 憲章の位置づけ

本憲章は、EA開発プロジェクトにおける 最上位の不変ルールを定義する。
本憲章に反する設計・実装・議論は一切行わない。

本憲章は、仕様・数値・実装手順・作業内容を定義しない。

1. 役割分離（最重要）

本プロジェクトは、以下の明確な役割分離を前提とする。

BASE EA

市場・時間・決済安全性を担保する共通基盤

ノータッチ派生される前提で設計・実装される

プロップ・ブローカー固有ルールを持たない

Compliance Layer

プロップ・運用ルールを実装する制御層

BASE EA を直接改変しない

BASE EA が公開する API のみを通じて制御を行う

2. 取引制御の原則

BASE EA は自律的な取引判断を行わない

取引の可否は外部から与えられる状態に従う

BASE EA は判断ではなく実行のみを担当する

3. 取引許可ゲート（必須原則）

BASE EA は外部制御可能な取引許可ゲートを必ず提供する

新規取引の可否は、このゲートを通じてのみ制御される

外部レイヤーは BASE EA の private / internal 実装に直接介入してはならない

4. 安全性優先原則

以下は、すべてのロジック・戦略・最適化に優先される。

決済の安全性

市場クローズ耐性

取引時間帯の安全性

5. 再議論禁止事項

以下は本憲章により再議論を禁止する。

BASE / Compliance の責務境界

ノータッチ派生前提

取引許可ゲート必須

内部実装への直接介入禁止

6. 下位文書との関係

本憲章はすべての仕様・引き継ぎ書・作業指示より優先される

下位文書は、本憲章に違反しない範囲でのみ有効とする

7. 未記載事項の扱い

本憲章に記載されていない事項は、未確定ではなく下位文書で定義される領域とする。
