2.  純粋憲章（Charter） EA開発プロジェクト 基本憲章（不変）

3.  憲章の位置づけ

本憲章は、EA開発プロジェクトにおける 最上位の不変ルールを定義する。
本憲章に反する設計・実装・議論は一切行わない。

本憲章は、仕様・数値・実装手順・作業内容を定義しない。

1.  役割分離（最重要）

本プロジェクトは、以下の明確な役割分離を前提とする。

Execution Layer

市場・時間・決済安全性を担保する実行層

注文実行 決済処理 例外処理 安全制御

Strategy Layer

売買判断を担当する戦略層

インジケーター判定 条件評価 エントリー判断 決済判断

Execution は自律的な取引判断を行わない。 Execution
は判断ではなく実行のみを担当する。

Strategy は Execution の private / internal
実装に直接介入してはならない。 Execution が公開する API
または制御経路のみを通じて動作する。

2.  取引制御の原則

Execution は自律的な取引判断を行わない。

取引の可否は Strategy および外部から与えられる状態に従う。

Execution は判断ではなく実行のみを担当する。

3.  取引許可ゲート（必須原則）

Execution は外部制御可能な取引許可ゲートを必ず提供する。

新規取引の可否は、このゲートを通じてのみ制御される。

Strategy はゲートを迂回してはならない。

内部実装への直接介入は禁止する。

4.  安全性優先原則

以下は、すべてのロジック・戦略・最適化に優先される。

決済の安全性 市場クローズ耐性 取引時間帯の安全性 異常終了耐性

5.  再議論禁止事項

以下は本憲章により再議論を禁止する。

Execution / Strategy の責務境界 取引許可ゲート必須
内部実装への直接介入禁止 安全性優先原則

6.  下位文書との関係

本憲章はすべての仕様・引き継ぎ書・作業指示より優先される。

下位文書は、本憲章に違反しない範囲でのみ有効とする。

7.  未記載事項の扱い

本憲章に記載されていない事項は、未確定ではなく下位文書で定義される領域とする。

8.  戦略可変前提条項

本部屋は検証前提の開発室である。

エントリー条件 決済条件 使用インジケーター パラメーター構成

これらは固定前提としない。

ただし、本憲章で定義される構造原則および安全原則は変動しない。
