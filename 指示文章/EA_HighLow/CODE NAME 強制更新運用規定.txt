CODE NAME 強制更新運用規定（正式追加ルール）
1. 目的

修正指示時に発生しやすい
「CODE NAME 更新忘れ」「旧prefix残留」「描画ID衝突」
を構造的に防止する。

本規定は、CODEXへ修正依頼を行うすべてのケースに適用される。

────────────────────────────────

2. 適用範囲（自動適用）

以下に該当する場合、本規定は自動適用される。

・仕様変更
・ロジック変更
・パラメータ追加/変更
・描画仕様変更
・状態遷移変更
・挙動に影響するあらゆる修正

※軽微であっても挙動に影響する場合は新CODE NAME必須。

────────────────────────────────

3. 冒頭固定ブロック義務（最重要）

CODEXに修正依頼を出す際、
必ず以下の「CODE NAME 強制更新ブロック」を最上部に貼ること。

順番変更・項目削除は禁止。

────────────────────────────────
■【CODE NAME 強制更新ブロック（必須・冒頭固定）】

1️⃣ 新CODE NAME
EA_HL_DEV_XXX_XXX

2️⃣ 旧CODE NAME
EA_HL_DEV_XXX_XXX

3️⃣ 変更理由（1行）

4️⃣ 旧prefix削除対象（直近2世代分明示）
・EA_HL_DEV_XXX_XXX.
・EA_HL_DEV_XXX_XXX.

5️⃣ CODE NAME 更新対象確認
□ private const string CODE_NAME 更新済
□ 描画IDは CODE_NAME + "." 形式
□ 固定文字列描画IDなし
□ ログに旧CODE NAME残留なし
□ ハードコード文字列残留なし

6️⃣ 描画ID衝突防止確認
□ REGIME_STATUS系 prefix化済
□ STOP_OVERLAY系 prefix化済
□ その他固定IDなし

7️⃣ Done条件確認
□ ログprefixに旧CODE NAMEが出ない
□ 描画prefixに旧CODE NAMEが出ない
□ CODE_NAME定数は1箇所のみ
□ 旧prefix削除が新CODE NAMEを誤削除していない

────────────────────────────────

────────────────────────────────

4. 強制原則

・このブロックが無い修正指示は無効
・修正内容より先に必ず記載する
・CODE NAMEを変更しない場合でも「変更なし」と明示する
・チェック項目は必ず全確認する

────────────────────────────────

5. 描画ID統一原則（固定）

・描画IDに固定文字列は禁止
・必ず CODE_NAME + "." + 固有名 形式を使用
・複数世代の同時起動を前提とする
・旧prefix削除は“直近2世代分のみ”を明示的に列挙

────────────────────────────────

6. 違反時の扱い

以下に該当する場合、修正は不合格とする。

・CODE NAME未更新
・旧prefix残留
・描画ID衝突
・ハードコードされた世代番号の残存

修正は再提出対象となる。

────────────────────────────────

以上を本プロジェクトにおける正式な運用固定規定とする。

今後、CODEXへ出すすべての修正指示は
この規定に従う。
